Introduction

Base URL: http://localhost:8000

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Auth APIs

User change password.

requires authentication

Within the config file, you are required to determine the number of previously used passwords a User is not allowed to use anymore by setting password.check_all to TRUE/FALSE or to an int value and password.number to a corresponding int value as well.

You can choose to notify a User whenever a password is changed by setting password.notify.change to TRUE

Example request:
curl --request POST \
    "http://localhost:8000/api/user/change/password" \
    --header "Authorization: Bearer {Your key}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"current_password\": \"Ex@m122p$%l6E\",
    \"password\": \"@wE3456qas@$\",
    \"password_confirmation\": \"@wE3456qas@$\"
}"

const url = new URL(
    "http://localhost:8000/api/user/change/password"
);

const headers = {
    "Authorization": "Bearer {Your key}",
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "current_password": "Ex@m122p$%l6E",
    "password": "@wE3456qas@$",
    "password_confirmation": "@wE3456qas@$"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (200):


{
"status": "success",
"status_code": 200,
"data": {
     "message": string
 }
}

Read user.

requires authentication

Fetch a record or records from the user table. The id param is optional but can either be a string|null|int

If the id has a null value the records will be paginated. The returned page size is be set from api.paginate.user_kyc.pageSize config.

If the id is a string value it can only be set as 'all'. This will return all the records without being paginated.

If the id value is an integer it will try to fetch a single matching record.

curl --request GET \
    --get "http://localhost:8000/api/user/users/read/all" \
